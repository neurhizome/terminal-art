## How It Works

The post-commit hook at `.git/hooks/post-commit` fires after every commit. If any `.md` file
in `docs/_posts/` or `docs/concepts/` changed, it runs `tools/graph_viz.py`.

`graph_viz.py` does three things:

1. **Parse** ‚Äî reads YAML frontmatter from every post and concept page. Extracts `title`, `date`, `tags`, `related`.
2. **Build** ‚Äî constructs a node graph. Nodes are typed (`session`, `concept`, `gradient`, `beginning`). Edges come from `related:` lists.
3. **Render** ‚Äî draws a 96√ó40 ANSI canvas with box-drawing characters: double-border for sessions, single for concepts, heavy-border for beginning, rounded for aesthetic posts. Edges are horizontal connectors with arrowheads, or L-shaped routed lines for cross-row links.

The output writes to `docs/assets/captures/knowledge-graph.ans` and is staged automatically (`git add`) so it appears in your next commit.

## Sleep Cycle

The idea: the graph doesn't just track what exists ‚Äî it eventually *compresses* it. When a cluster of tightly-linked nodes builds up, the graph could fold the cluster into a topological knot: a single node that represents the attractor the cluster has converged to. This is the "interference pattern" interpretation ‚Äî the knowledge graph runs its own kind of diffusion, and high-density regions collapse into concepts.

That compression pass isn't implemented yet. But the infrastructure is here.
